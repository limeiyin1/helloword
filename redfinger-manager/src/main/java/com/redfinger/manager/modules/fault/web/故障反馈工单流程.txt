1. 故障反馈处理流程
	    a.新建故障反馈工单(save)
		     aa.新建故障反馈工单有三种方式
				1)后台新建(故障来源为:win)
				2)用户新建
				    若用户的故障反馈工单数量>=5, 则不能新建故障反馈工单
				3)系统新建(故障来源为:system)
				
		b.受理故障反馈工单(accept)
		      aa.用户只能受理故障状态故障工单状态为: 
			       1)0(新工单)
				   2)2(移交客服)
				   3)3(移交测试)
				   4)4(移交运维)
			  bb.故障工单受理完成后, 其故障状态会设置为: 1(处理中)
			  cc.受理完成后, 设备的故障状态设置为: 1(故障)
			  dd.设置最后处理人为受理用户
			  ee.标志故障反馈工单的'是否解决'为:0(未解决)
			  
		c.处理故障反馈工单(fix)
			  aa.只能处理故障反馈工单状态为: 1(处理中)
              bb.最后处理人(即谁受理的工单就必须由谁处理)必须为当前用户才能对故障工单进行处理
			  cc.处理流程
			     处理方式:
					1)移交客服 || 移交测试
					    故障反馈工单的故障状态重新设置为:1(处理中), 故障反馈工单流程重新进入b步骤
					2)移交运维
					    a1.设置修复类型
						a2.设置故障修复内容
						a3.故障反馈工单的故障状态重新设置为:1(处理中), 故障反馈工单流程重新进入b步骤
					3)已处理
						a1.设置修复类型
						a2.设置故障修复内容
						a3.标志故障反馈工单的'是否解决'为:1(已解决)
					    a4.故障工单处理完成, 进入故障'处理完毕确认流程'
			  
		d.处理完毕确认(reply)
		      aa.只能确认故障反馈工单状态为:5(已处理)
			  bb.查询所有故障反馈工单, 如若所有故障反馈工单已经处理完毕, 则把设备的'故障状态'设置为: 0(正常)
			  cc.设置故障工单反馈状态为:　9(完结)
			  
		e.完结
